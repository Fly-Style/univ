    1) RUP - итеративный подход к построению тестирования, выпуска версий продукта
        в соответствии к решению проблем, учета рисков и выпуска проекта настолько быстро, насколько это возможно.

        RUP разделяет один цикл на четыре последовательныъ стадии - начало, уточнение, конструирование, выпуск.
        // TODO : watch RUP process.

    2) Вся разработка CIS - процесс создания артефактов.
        Артефакт - это финальный/промежуточный продукт, который изготавливается и юзается во время разработки
        проекта. Например, это может быть :
            - документы (документированный поток событий);
            - модель (концептуальная модель, логико-алгоритмическая модель, физическая модель);
            - элемент модели;
            - исходный код.

    3)  Процесс разработки.
        а. Начало - определение бизнес-модели проекта, согласование плана разработки проекта. Нередко создается прототип.
        b. Уточнение - ислледование, уточннение плана и разработка архитектуры проекта, выбор архитектуры системы,
            создание. Главное на этом этапе - принятие решения о начале разработки самой системы.
        с. Конструирование - поэтапное создание CIS. По окончании этой стадии принимается решение
            про готовность программного продукта к эксплуатационных площадок и внедрение пользователям.
        d. Внедрение CIS конечным пользователям.

        В конце КАЖДОЙ фазы принимается решение о переходе к следующей стадии проекта



UML.
    Главный принцип - абстрагирование, формализация данных. Второй принцип - многомодельность.
    Многоиерахичность построения моделей сложных систем, тобишь обзор процесса моделирования на разных уровнях
    абстракции  и детализации.

    Типы диаграмм :
        1) Диаграмма вариантов использования [Use Case]
        2) Диаграмма классов
        3) Группа диаграмм поведения [State, Activity, Interaction, Sequence, Collaboration]
        4) Диаграммы реализации [Components, Deployment ... ]\

    Контент даиграмм : графические символы, текст, связи
